# JARVIS V2

**JARVIS V2** is a hybrid AI agent architecture that combines the power of cloud reasoning (via Google Gemini) with the privacy and execution capabilities of a local AI fleet (via Ollama).

It acts as an autonomous assistant capable of executing scripts, managing local memory, running headless browsing tasks, and orchestrating a local fleet of specialized LLMs—all controlled from an Electron desktop application or via WhatsApp (using the OpenClaw Gateway).

## Key Features

- **Hybrid Orchestration**: Uses a fast cloud model (dynamically configured, defaults to Gemini 1.5 Pro or Gemini 2.5 Flash) as a "Librarian" to route tasks to specialized local models.
- **Background Task Manager (Daemons)**: Proactively run infinite loops or monitoring scripts (like USB watchers or download monitors) in the background. Stream their live output directly to the UI, track their PIDs, and kill them at will.
- **Local AI Fleet**: Automatically provisions and manages local Ollama models tailored to your hardware (e.g., `gemma` for reasoning, `qwen` for coding, `hermes` for OS tasks).
- **Persistent Local Memory**: Features a long-term memory vault and RAG (Retrieval-Augmented Generation) system. Memories are stored locally, continuously consolidated offline, and never sent to the cloud.
- **Automation Vault**: Executes Python/PowerShell scripts, controls browsers, and searches the web autonomously. Scripts are dynamically indexed so JARVIS knows what tools it has at its disposal.
- **Transparent Reasoning**: The raw JSON output generated by the Cloud Orchestrator and Planner is automatically logged to the file system (`vault/cloud_logs/`) so you can inspect its exact routing and task breakdown logic.
- **WhatsApp Gateway (OpenClaw)**: A built-in traffic master for routing WhatsApp messages to the local agent.
- **Desktop UI**: A rich React-like dashboard built on Electron for real-time system telemetry, agent communication, daemon management, and model warmth tracking.

## Getting Started

Please see the [SETUP.md](./SETUP.md) for full prerequisites and an onboarding guide.

### Quick Start
1. Ensure Node.js and Ollama are installed.
2. Run the automated setup script in PowerShell:
   ```powershell
   .\setup.ps1
   ```
3. Start the Agent:
   ```bash
   npm start
   ```

## Architecture

- `src/api/` - Express HTTP + WebSocket server
- `src/orchestrator/` - Gemini cloud router and planner
- `src/fleet/` - Local Ollama model management and capability routing
- `src/daemon/` - Background Task Manager for long-running scripts
- `src/memory/` - Long-term memory & RAG
- `src/tools/` - Tool executor (scripts, web search, desktop notifications)
- `src/gateway/openclaw/` - WhatsApp gateway
- `src/desktop/` - Electron entry point
- `public/` - Frontend dashboard
- `vault/` - Persistent local data (config, logs, memory, conversations)
